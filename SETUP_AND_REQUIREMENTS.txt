==============================================================================
 SOCIAL TREND FINDER — SETUP, REQUIREMENTS & TESTING GUIDE
==============================================================================
 Architecture: a Claude skill. Claude is the host and writes the summaries
 itself (no AI API key). Trend data is fetched centrally by a GitHub Action,
 so the skill needs no keys to fetch. The only per-user credential is a
 SendGrid key for emailing the digest to the team.


------------------------------------------------------------------------------
 0. THE TWO SIDES
------------------------------------------------------------------------------
   SERVER SIDE (runs in GitHub Actions, owned by an admin)
     generate_feed.py + instagram.py + tiktok.py + scorer.py +
     base_fetcher.py + settings.py
     -> fetches Instagram + TikTok, scores by engagement velocity, and
        commits feed-instagram.json / feed-tiktok.json to the repo.
     -> needs the Meta + TikTok keys, stored as GitHub Secrets.

   SKILL SIDE (runs inside Claude, on each user's machine)
     prepare_digest.py  -> reads the committed feeds + prompts + config (HTTP)
     deliver.py         -> emails the finished digest via SendGrid
     prompts/*.md       -> plain-English summary instructions
     -> uses ONLY the Python standard library (no pip install)
     -> needs ONLY a SendGrid key (for the person sending), stored locally.

   API KEY COUNT:  4 (old design)  ->  1 per user (SendGrid)  +  fetch keys
                   held once, centrally, by the admin.


------------------------------------------------------------------------------
 1. SKILL SIDE — what a user needs
------------------------------------------------------------------------------
   Prerequisites:
     - Python 3.x (standard library only — nothing to pip install)
     - The skill installed at ~/.claude/skills/social-trend-finder
     - A SendGrid API key + a verified sender address

   Local files the skill creates during onboarding:
     ~/.social-trend-finder/.env          -> SENDGRID_API_KEY=...
     ~/.social-trend-finder/config.json   -> recipients, fromEmail, frequency,
                                             language, onboardingComplete
     (optional) ~/.social-trend-finder/prompts/<file>.md
                                          -> per-user prompt overrides

   config.json fields (see config/config.example.json):
     recipients   list of company email addresses to send to
     fromEmail    a SendGrid-verified sender address
     frequency    "weekly" (default) or "daily" — informational
     language     "en" | "zh" | "bilingual"

   Run a digest (what Claude does for you):
     python3 prepare_digest.py            # prints feed+prompts+config as JSON
     ...Claude writes the HTML digest from that JSON...
     python3 deliver.py --subject "..." --file /tmp/stf-digest.html


------------------------------------------------------------------------------
 2. SERVER SIDE — what the admin needs (one-time)
------------------------------------------------------------------------------
   Python dependencies (requirements.txt) — installed automatically by the
   GitHub Action; only needed if you run the fetcher locally:
     requests==2.32.3      Instagram token refresh
     scrapling             web-fetch backend for base_fetcher.safe_get
     tenacity==8.3.0       retry/backoff on rate limits
     python-dotenv==1.0.1  load .env for local fetch testing

   GitHub repository SECRETS (Settings -> Secrets and variables -> Actions):
     INSTAGRAM_ACCESS_TOKEN   long-lived IG token (60-day expiry)
     INSTAGRAM_USER_ID        IG Business/Creator user ID
     ENSEMBLEDATA_API_TOKEN   EnsembleData (covers IG + TikTok)   } at least
        -- or --                                                  } one of
     LAMATOK_API_KEY          LamaTok (TikTok-only, cheaper)      } these

   GitHub repository VARIABLES:
     HASHTAGS_TO_TRACK        e.g. skincare,aestheticfashion,cleanbeauty
     TIKTOK_REGIONS           e.g. GB,US

   First run:
     Actions tab -> "Generate Trend Feeds" -> Run workflow.
     This creates feed-instagram.json / feed-tiktok.json. After that the
     workflow runs on the schedule in .github/workflows/generate-feed.yml
     (default: weekly, Monday 06:00 UTC).

   If you fork to a different repo, set STF_FEED_BASE (raw URL base) so the
   skill's prepare_digest.py reads YOUR feeds. Default points at
   https://raw.githubusercontent.com/kweeyo09/social-trend-finder/main


------------------------------------------------------------------------------
 3. WHERE TO GET THE KEYS (admin side) + GOTCHAS
------------------------------------------------------------------------------
   META (Instagram Graph API)   developers.facebook.com -> create app
     - Business/Creator accounts only; 30 hashtag searches/week per account;
       long-lived token expires after 60 days.
   TIKTOK (third-party)
     - EnsembleData ($) covers IG + TikTok; LamaTok ($, cheaper, 100 free).
       The official TikTok API does not expose trending data.
   SENDGRID (skill side)         app.sendgrid.com -> API Keys
     - fromEmail must be a verified sender/domain.


------------------------------------------------------------------------------
 4. TESTING
------------------------------------------------------------------------------
   Skill side (no keys needed):
     - python3 prepare_digest.py
       Before the Action has run, the feeds are placeholders (empty arrays) so
       you'll see stats of 0 and a note in "errors" — that's expected.
     - deliver.py dry run: with no .env/config, it PRINTS the digest instead of
       sending, so you can test formatting safely:
         echo "<h1>hi</h1>" | python3 deliver.py --subject "test"
     - Pure-logic modules (server side) can be unit-tested with fake dicts:
         scorer.rank_tiktok_sounds([...]) / scorer.score_instagram_post({...})

   Server side (needs keys):
     pip install -r requirements.txt
     export INSTAGRAM_ACCESS_TOKEN=... INSTAGRAM_USER_ID=... LAMATOK_API_KEY=...
     export HASHTAGS_TO_TRACK=skincare,cleanbeauty TIKTOK_REGIONS=GB,US
     python generate_feed.py        # writes the two feed-*.json files


------------------------------------------------------------------------------
 5. FILE MAP
------------------------------------------------------------------------------
   SKILL.md            The skill Claude runs (onboarding + digest workflow).
   README.md           Human quick-start (user + admin).
   prepare_digest.py   Skill side: feed + prompts + config -> JSON (stdlib).
   deliver.py          Skill side: email via SendGrid HTTP API (stdlib).
   prompts/*.md        Editable summary instructions.
   generate_feed.py    Server side: fetch + score -> feed JSON.
   instagram.py        Server side: Meta Graph API fetchers.
   tiktok.py           Server side: EnsembleData / LamaTok fetchers.
   scorer.py           Server side: engagement-velocity scoring (pure).
   base_fetcher.py     Server side: Scrapling fetch + tenacity retry.
   settings.py         Server side: validates fetch env vars.
   config/config.example.json   Template for ~/.social-trend-finder/config.json
   feed-instagram.json / feed-tiktok.json   Committed feeds (Action output).
   .github/workflows/generate-feed.yml      The server-side cron job.

   REMOVED in this architecture: main.py, scheduler.py, summariser.py,
   sender.py, renderer.py (replaced by the central feed + Claude-as-host +
   prepare_digest.py / deliver.py).
