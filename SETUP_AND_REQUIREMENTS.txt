==============================================================================
 SOCIAL TREND FINDER — SETUP, REQUIREMENTS & TESTING GUIDE
==============================================================================
 Architecture: a Claude skill. Claude is the host and writes the summaries
 itself (no AI API key). Trend data is fetched centrally by a GitHub Action,
 so the skill needs no keys to fetch. The only per-user credential is a
 SendGrid key for emailing the digest to the team.

 Platforms covered: TikTok, Instagram (Reels), YouTube (Shorts + long-form),
 and Reddit. TikTok + Instagram run through ONE paid provider (ScrapeCreators);
 YouTube (yt-dlp) and Reddit (public JSON) are KEYLESS.


------------------------------------------------------------------------------
 0. THE TWO SIDES
------------------------------------------------------------------------------
   SERVER SIDE (runs in GitHub Actions, owned by an admin)
     generate_feed.py + instagram.py + tiktok.py + reddit.py + youtube.py +
     scorer.py + base_fetcher.py + settings.py
     -> fetches all four platforms, scores by engagement velocity, and commits
        feed-instagram.json / feed-tiktok.json / feed-reddit.json /
        feed-youtube.json to the repo.
     -> needs ONE key: SCRAPECREATORS_API_KEY (TikTok + Instagram), stored as a
        GitHub Secret. YouTube + Reddit need no key.

   SKILL SIDE (runs inside Claude, on each user's machine)
     prepare_digest.py  -> reads the committed feeds + prompts + config (HTTP)
     deliver.py         -> emails the finished digest via SendGrid
     prompts/*.md       -> plain-English summary instructions
     -> uses ONLY the Python standard library (no pip install)
     -> needs ONLY a SendGrid key (for the person sending), stored locally.

   API KEY COUNT:  1 central (ScrapeCreators, admin)  +  1 per sender (SendGrid).
                   YouTube and Reddit are free/keyless.


------------------------------------------------------------------------------
 1. SKILL SIDE — what a user needs
------------------------------------------------------------------------------
   Prerequisites:
     - Python 3.x (standard library only — nothing to pip install)
     - The skill installed at ~/.claude/skills/social-trend-finder
     - A SendGrid API key + a verified sender address

   Local files the skill creates during onboarding:
     ~/.social-trend-finder/.env          -> SENDGRID_API_KEY=...
     ~/.social-trend-finder/config.json   -> recipients, fromEmail, frequency,
                                             language, onboardingComplete
     (optional) ~/.social-trend-finder/prompts/<file>.md
                                          -> per-user prompt overrides

   config.json fields (see config/config.example.json):
     recipients   list of company email addresses to send to
     fromEmail    a SendGrid-verified sender address
     frequency    "weekly" (default) or "daily" — informational
     language     "en" | "zh" | "bilingual"

   Run a digest (what Claude does for you):
     python3 prepare_digest.py            # prints feeds+prompts+config as JSON
     ...Claude writes the HTML digest from that JSON...
     python3 deliver.py --subject "..." --file /tmp/stf-digest.html


------------------------------------------------------------------------------
 2. SERVER SIDE — what the admin needs (one-time)
------------------------------------------------------------------------------
   Python dependencies (requirements.txt) — installed automatically by the
   GitHub Action; only needed if you run the fetcher locally:
     scrapling             web-fetch backend for base_fetcher.safe_get
     tenacity==8.3.0       retry/backoff on rate limits
     python-dotenv==1.0.1  load .env for local fetch testing
     yt-dlp                keyless YouTube search + metadata

   GitHub repository SECRETS (Settings -> Secrets and variables -> Actions):
     SCRAPECREATORS_API_KEY   one key, covers TikTok + Instagram
                              (scrapecreators.com — 100 free credits, then
                              pay-as-you-go; sent as the x-api-key header)

   GitHub repository VARIABLES (all optional — sensible defaults exist):
     HASHTAGS_TO_TRACK   Instagram reel-search terms, e.g. skincare,cleanbeauty
     TIKTOK_COUNTRIES    popular-hashtags country codes, e.g. GB,US
     TIKTOK_PERIOD       7, 30, or 120 (days)
     TIKTOK_INDUSTRY     e.g. beauty-and-personal-care (blank = all)
     IG_DATE_POSTED      last-hour|last-day|last-week|last-month|last-year
     REDDIT_SUBREDDITS   e.g. SkincareAddiction,beauty,femalefashionadvice
     REDDIT_QUERIES      optional site-wide keyword searches
     REDDIT_PERIOD       hour|day|week|month|year|all
     YOUTUBE_QUERIES     e.g. skincare routine,outfit ideas
     YOUTUBE_RECENT_DAYS only keep videos newer than this (default 30)

   First run:
     Actions tab -> "Generate Trend Feeds" -> Run workflow.
     This creates the four feed-*.json files. After that the workflow runs on
     the schedule in .github/workflows/generate-feed.yml (default: weekly,
     Monday 06:00 UTC).

   If you fork to a different repo, set STF_FEED_BASE (raw URL base) so the
   skill's prepare_digest.py reads YOUR feeds. Default points at
   https://raw.githubusercontent.com/kweeyo09/social-trend-finder/main


------------------------------------------------------------------------------
 3. WHERE TO GET THE KEYS (admin side) + GOTCHAS
------------------------------------------------------------------------------
   SCRAPECREATORS (TikTok + Instagram)   scrapecreators.com -> sign up -> API key
     - One key covers both platforms. Each request costs ~1 credit; 100 free to
       start. Popular-hashtags is fully documented; trending SOUNDS are derived
       best-effort from the popular-videos feed and may be empty on some runs.
   YOUTUBE  — keyless. yt-dlp searches + reads public metadata. No account.
   REDDIT   — keyless. Public .json endpoints with a browser User-Agent.
   SENDGRID (skill side)         app.sendgrid.com -> API Keys
     - fromEmail must be a verified sender/domain.


------------------------------------------------------------------------------
 4. TESTING
------------------------------------------------------------------------------
   Skill side (no keys needed):
     - python3 prepare_digest.py
       Before the Action has run, the feeds are placeholders (empty arrays) so
       you'll see stats of 0 and a note in "errors" — that's expected.
     - deliver.py dry run: with no .env/config, it PRINTS the digest instead of
       sending, so you can test formatting safely:
         echo "<h1>hi</h1>" | python3 deliver.py --subject "test"
     - Pure-logic modules (server side) can be unit-tested with fake dicts:
         scorer.rank_reddit_posts([...]) / scorer.rank_instagram_reels([...])

   Server side (needs the ScrapeCreators key for IG/TikTok only):
     pip install -r requirements.txt
     export SCRAPECREATORS_API_KEY=...
     export HASHTAGS_TO_TRACK=skincare,cleanbeauty TIKTOK_COUNTRIES=GB,US
     python generate_feed.py                 # all four feeds
     python generate_feed.py --reddit-only   # keyless: no ScrapeCreators key needed*
     python generate_feed.py --youtube-only  # keyless: no ScrapeCreators key needed*
     (* reddit.py / youtube.py read their own env and don't import settings,
        so the keyless feeds build even without the ScrapeCreators key.)


------------------------------------------------------------------------------
 5. FILE MAP
------------------------------------------------------------------------------
   SKILL.md            The skill Claude runs (onboarding + digest workflow).
   README.md           Human quick-start (user + admin).
   prepare_digest.py   Skill side: feeds + prompts + config -> JSON (stdlib).
   deliver.py          Skill side: email via SendGrid HTTP API (stdlib).
   prompts/*.md        Editable summary instructions (one per platform).
   generate_feed.py    Server side: fetch + score -> feed JSON.
   instagram.py        Server side: ScrapeCreators Instagram Reels fetcher.
   tiktok.py           Server side: ScrapeCreators TikTok hashtags + sounds.
   reddit.py           Server side: keyless Reddit public-JSON fetcher.
   youtube.py          Server side: keyless yt-dlp search (Shorts + long-form).
   scorer.py           Server side: engagement-velocity scoring + caps/normalise.
   base_fetcher.py     Server side: Scrapling fetch + tenacity retry.
   settings.py         Server side: validates the ScrapeCreators key + options.
   config/config.example.json   Template for ~/.social-trend-finder/config.json
   feed-*.json         Committed feeds (Action output): instagram, tiktok,
                       reddit, youtube.
   .github/workflows/generate-feed.yml      The server-side cron job.

   REMOVED in this architecture: main.py, scheduler.py, summariser.py,
   sender.py, renderer.py (replaced by the central feed + Claude-as-host +
   prepare_digest.py / deliver.py).
