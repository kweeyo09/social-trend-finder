==============================================================================
 SOCIAL TREND FINDER — SETUP, REQUIREMENTS & TESTING GUIDE
==============================================================================
 What this covers: every dependency, environment variable, and API key the
 skill needs, plus exactly what to do to edit and test it locally.

 NOTE: This is a Claude *skill* (SKILL.md is the entry point), but it runs by
 executing the bundled Python scripts. So "make it run" = make the Python run.


------------------------------------------------------------------------------
 0. CURRENT STATUS — READ FIRST
------------------------------------------------------------------------------
 The two original code-level blockers have now been FIXED:

   (A) base_fetcher.py — NOW CREATED.
       Provides safe_get(url, params, headers, timeout) -> dict.
       It fetches via the Scrapling library (scrapling.fetchers.Fetcher)
       and is wrapped in tenacity for retry/backoff on HTTP 429 and 5xx.
       All endpoints return JSON, so safe_get returns the parsed JSON body.

   (B) Imports — NOW FLATTENED.
       Chosen for stability: every file sits in the project root and imports
       by plain module name (e.g. `import settings`, `from instagram import`,
       `from base_fetcher import safe_get`). No src/ or config/ folders and no
       __init__.py needed. Run from the project root and imports always
       resolve. (NOTE: SKILL.md still *describes* a src/ tree — that diagram is
       now out of date vs. the actual flat layout.)

 What remains before a real run: install dependencies (section 2) and supply
 credentials (sections 3-4). The code compiles and the import graph is
 consistent; it has NOT been run end-to-end here because no deps were installed
 in the editing environment.

 3. ENVIRONMENT VARIABLES  (create a .env file in project root)
------------------------------------------------------------------------------
 settings.py loads these via python-dotenv and HARD-FAILS at startup if any
 "REQUIRED" var is missing. Copy .env.example -> .env, then fill in.

 REQUIRED
 --------
   META_APP_ID             Meta app ID
   META_APP_SECRET         Meta app secret
   INSTAGRAM_ACCESS_TOKEN  Long-lived IG token (60-day expiry)
   INSTAGRAM_USER_ID       Your IG Business/Creator user ID
   SENDGRID_API_KEY        SendGrid API key
   EMAIL_FROM              Verified SendGrid sender address
   EMAIL_TO                Recipient(s), comma-separated
   ANTHROPIC_API_KEY       Claude API key

   TikTok provider — AT LEAST ONE of these is required
   (settings.py raises an error if both are empty):
     ENSEMBLEDATA_API_TOKEN   EnsembleData (covers IG + TikTok)
     LAMATOK_API_KEY          LamaTok (TikTok-only, cheaper)
   If ENSEMBLEDATA_API_TOKEN is set it is used; otherwise LamaTok.

 OPTIONAL (have defaults)
 ------------------------
   INSTAGRAM_API_VERSION    default: v21.0
   HASHTAGS_TO_TRACK        default: skincare,aestheticfashion
   TIKTOK_REGIONS           default: GB,US
   TIMEZONE                 default: Europe/London
   LOG_FILE                 default: data/trends_log.json
   REPORT_SCHEDULE          documented but NOT actually read by scheduler.py
                            (the Mon/Wed/Fri 08:00 cron is hardcoded)


------------------------------------------------------------------------------
 4. API KEYS — WHERE TO GET THEM & GOTCHAS
------------------------------------------------------------------------------
   META (Instagram Graph API)        developers.facebook.com -> create app
     - Free, but requires OAuth app review.
     - Works only on Business/Creator accounts you control.
     - Hashtag search capped at 30 unique hashtags/week per account.
     - Rate limit ~200 requests/hour.
     - Long-lived token expires after 60 days. instagram.py has a
       refresh_token() helper, but nothing calls it automatically yet.
     - Insights features need 1,000+ followers.

   TIKTOK (third-party — pick one)
     - EnsembleData   ensembledata.com    (~$100-$1,400/mo; IG + TikTok)
     - LamaTok        api.lamatok.com      (~$0.0006/req; 100 free to trial)
     - Official TikTok API does NOT expose trending sounds/hashtags, so a
       third-party provider is mandatory for TikTok data.

   SENDGRID                            app.sendgrid.com -> API Keys
     - EMAIL_FROM must be a verified sender/domain.

   ANTHROPIC (Claude)                  console.anthropic.com
     - summariser.py calls model "claude-sonnet-4-6".

 5. HOW TO EDIT & TEST LOCALLY
------------------------------------------------------------------------------
 STEP 1 — Environment
   python -m venv .venv
   source .venv/bin/activate            (Windows: .venv\Scripts\activate)
   pip install -r requirements.txt

 STEP 2 — (Already done — see section 0. base_fetcher.py exists and imports
          are flattened. Nothing to do here unless you change the layout.)
   NOTE: Scrapling's plain Fetcher is pure HTTP and needs no browser install.
   Only StealthyFetcher/DynamicFetcher would need `scrapling install`; this
   project does not use those.

 STEP 3 — Credentials
   cp .env.example .env
   # edit .env, fill in section 3 values
   # for cheapest testing: use LAMATOK_API_KEY (100 free requests)

 STEP 4 — One-off test run
   python main.py --once
   # runs FETCH -> SCORE -> SUMMARISE -> EMAIL -> LOG once and exits.

 STEP 5 — Test the scheduler (optional)
   python scheduler.py
   # blocks and runs the job Mon/Wed/Fri 08:00 in TIMEZONE. Ctrl+C to stop.

 STEP 6 — Check output
   - Email arrives at EMAIL_TO addresses.
   - Run metadata appended to data/trends_log.json (created automatically).

 TESTING TIPS (to avoid burning API quota / sending real emails while editing)
   - Test scoring in isolation — scorer.py is pure functions, no network:
       from scorer import score_instagram_post, rank_tiktok_sounds
     Feed it fake dicts and check the numbers.
   - Test the email HTML without sending — call render_digest() in renderer.py
     with sample data and write the returned string to a .html file to preview.
   - Comment out / stub send_digest() in sender.py so no real email goes out.
   - Stub the fetchers to return canned JSON so you don't hit Meta/TikTok at
     all while iterating on scoring or rendering.
   - Watch the console: logging is INFO level, so each FETCH/SCORE/EMAIL step
     prints progress and errors.
